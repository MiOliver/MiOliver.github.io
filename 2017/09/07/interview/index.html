Java 服务端研发面试总结

Posted on 2017-09-07 |

笔者要换个城市生活了，因为大北京最近一直下雨，我受不了了（实际上是房价太特么的高了，高的我买不起，现在的钱倒是能买个厕所），回归正题，说一下Java工程师的面试经验。

Java 基础

  • JVM 相关
  1. jvm的内存模型、不同区域的功能。垃圾回收过程、回收策略方式、jvm的优化，经常使用的优化参数代表的意思是什么。
  • JDK 源码
  1. hashMap的源码，底层实现原理，代码细节，明白冲突的处理过程
  2. 不同类型的变量代表的意思
  3. 异常，以及异常的处理流程。
  • 高并发相关
  1. concurrent包里的并发工具类，了解源码和使用场景;
  2. 线程的实现，已经不同线程之间的同步方式都有哪些，相互之间的差异是什么。
  3. 能否自己使用JDK最基础的工具实现多线程的互斥和写作，典型的生产者消费者。
  4. 了解Java线程池的实现原理，相关代码细节

项目相关

  1. 自己做过的项目一定要总结，找到亮点和挑战点，面试官最通常想知道的不是你做的项目具体是做什么，他关注的通常是你遇到的有挑战性的问题，以及你是怎么解决的。
  2. 针对项目，面试官通常是由你项目的中东西，慢慢的深入或者发散，提出更有挑战性的题目，看你究竟能了解多少，所以不要企图欺骗他，即使不是你做的项目，但是你把这个项目搞懂了，技术点和难点都明白，基本这个项目就可以为你所用了。

算法和数据结构

  1. 手写代码，这是大多数公司在前两轮面试中都会使用的面试手段，搞技术的离不开写代码，手写代码的难易程度不尽相同，但是如果你连最基本的排序和相关的算法复杂度都不清楚，基本上通过的概率不会高了。如果你写不出来，可以坦白，但是最好告诉下面试官你的解题思路。
  2. 基本的数据结构，链表、数组、栈、树、堆、图等，还是需要知道的

数据库

  1. 这个也算是Java web 工程师必备的技能之一吧，熟悉sql的增删改查
  2. sql 表的设计，以及设计的时候的原则，设计过程中就应该考虑查询的效率问题。
  3. sql的执行过程，以及如何根据sql的执行过程，选择性提出优化思路。

大数据相关

  1. mongoDB redis hbase hive
  2. hadoop 相关
  3. spark 离线数据统计

书单推荐

  2. 《Java并发编程实战》
  3. 《java多线程编程核心技术》
  4. 《Effective Java中文版》
  5. 《大型网站技术架构 核心原理与案例分析》
  6. 《大型网站系统与Java中间件实践》
  7. 《从Paxos到ZooKeeper 分布式一致性原理与实践》
  8. 《Spring源码深度解析》
  9. 《Spark 快速大数据分析》、
  10. 《亿级流量网站架构和实践》
